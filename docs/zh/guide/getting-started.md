# 快速开始

本文介绍 EveryoneTrust SSL 中最常见的证书申请流程。

## 基本流程

| 步骤 | 你需要做什么 | 结果 |
| --- | --- | --- |
| 1 | 选择证书产品 | 产品匹配域名类型和验证等级。 |
| 2 | 提交订单 | 系统创建证书申请。 |
| 3 | 完成域名验证 | CA 确认你拥有域名控制权。 |
| 4 | 部署证书 | 网站开始使用有效 HTTPS 证书链。 |
| 5 | 监控与续期 | 避免证书意外过期。 |

::: tip
如果你能管理域名 DNS，建议先配置 DNS 授权。后续签发、重签和续期都会更顺畅。
:::

## 下单前准备

- 确认主域名，例如 `example.com` 或 `*.example.com`。
- 判断是否需要单域名、通配符、多域名、OV、EV 或 IP 证书。
- 准备验证方式，生产场景优先选择 DNS/CNAME。
- 明确私钥由谁生成和保管。
